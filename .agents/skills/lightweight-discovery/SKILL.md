# Lightweight Discovery

## Purpose

Use this skill when the request is unclear, broad, risky, or likely to affect multiple areas. Keep discovery short and action-oriented.

## Rules

- Ask no questions when a reasonable best-effort path is available.
- Ask at most 3 clarifying questions when missing information would materially change the result.
- Do not run a long brainstorming interview.
- Prefer repository evidence over speculation.
- End with a brief that can feed `phase-planner` or direct implementation.

## Output

Write a compact brief with:

- Goal
- Known constraints
- Assumptions
- Open questions, if any
- Recommended next step
