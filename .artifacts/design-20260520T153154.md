# Design: 젤리서랍 재구현

**Linked requirements:** requirements-20260520T153154.md

## 접근 방식

기존 MVVM 레이어(JellyViewModel, JellyRepository)는 변경 없이 재사용한다.  
변경 범위: Activity, Adapter, 레이아웃 XML, 신규 Detail Activity.

---

## 변경 파일

### 1. `JellyDrawerActivity.java` (수정)
- `LinearLayoutManager` → `GridLayoutManager(this, 3)` 교체
- `adapter.setOnStartAgingListener(...)` 콜백 제거
- `adapter.setOnItemClickListener(jellyId → JellyDetailActivity 진입)` 추가
- `JellyViewModel.isLoading` observe → ProgressBar 토글 유지
- `getJellyList(userId)` observe 유지 (빈 상태 처리 유지)

### 2. `JellyDrawerAdapter.java` (수정)
- `OnStartAgingListener` 인터페이스 및 `btnStartAging` 관련 코드 전부 제거
- 숙성 상태 배지(tvAgingStatus) 바인딩 제거
- `OnItemClickListener` 인터페이스 추가 (jellyId 전달)
- `itemView.setOnClickListener` → `onItemClickListener.onItemClick(jellyId)` 호출
- `serverUrl` 필드 제거 → `RetrofitClient.BASE_URL` 참조

### 3. `item_jelly_drawer.xml` (수정)
- 정사각형에 가까운 컴팩트한 카드 레이아웃으로 재설계
- 상단: 두 감정 아이콘 (emo1ImageView, emo2ImageView) 나란히 배치
- 하단: 날짜 텍스트 (tvCreateDate)
- 숙성 배지(tvAgingStatus)·숙성 버튼(btnStartAging) 제거

### 4. `JellyDetailActivity.java` (신규)
- Intent Extra로 `jellyId: Long` 수신
- `JellyViewModel.getJellyById(jellyId)` 호출
- `JellyResDTO` 데이터 바인딩: jellyName, content, createDate
- ViewBinding 사용 (`ActivityJellyDetailBinding`)

### 5. `activity_jelly_detail.xml` (신규)
- ConstraintLayout
- tvJellyName, tvCreateDate, tvContent

### 6. `AndroidManifest.xml` (수정)
- `JellyDetailActivity` 등록

---

## 데이터 흐름

```
JellyDrawerActivity
  → GridLayoutManager(3) RecyclerView
  → jellyViewModel.getJellyList(userId)
    → JellyRepository.getJellyList(userId)
      → GET /jelly/user/{userId}
    ← LiveData<Resource<List<JellyDrawerResDTO>>>
  ← adapter.submitList(list)

아이템 탭 →
JellyDetailActivity (jellyId via Intent)
  → jellyViewModel.getJellyById(jellyId)
    → JellyRepository.getJellyById(jellyId)
      → GET /jelly/{jellyId}
    ← LiveData<Resource<JellyResDTO>>
  ← UI 바인딩 (jellyName, content, createDate)
```

---

## 재사용 (변경 없음)

| 파일 | 이유 |
|------|------|
| `JellyViewModel.java` | getJellyList, getJellyById 이미 올바르게 구현됨 |
| `JellyRepository.java` | Context 기반 Retrofit 사용, 인증 포함 |
| `JellyDrawerResDTO.java` | 필요한 필드(emo1Icon, emo2Icon, createDate) 포함 |
| `JellyResDTO.java` | 상세 화면에 필요한 필드 포함 |
| `SessionManager.java` | userId 획득 경로 변경 없음 |

---

## 구현 순서

1. `item_jelly_drawer.xml` 레이아웃 재설계 (그리드 카드)
2. `JellyDrawerAdapter` 단순화 (OnItemClickListener, serverUrl 제거)
3. `JellyDrawerActivity` 수정 (GridLayoutManager, 클릭 리스너)
4. `activity_jelly_detail.xml` + `JellyDetailActivity` 신규 생성
5. `AndroidManifest.xml` JellyDetailActivity 등록
