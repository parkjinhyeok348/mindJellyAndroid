# Requirements: 젤리서랍 재구현

**Created:** 2026-05-20T15:31:54  
**Phase:** brainstorm

---

## 목적

로그인한 사용자가 '오늘의 마음젤리'에서 작성한 일기 목록을 3열 그리드로 조회하고, 항목을 탭하면 해당 일기 상세 화면으로 이동할 수 있는 젤리서랍 화면을 재구현한다.

## 사용자

로그인 상태의 앱 사용자 (userId는 SessionManager에서 JWT 파싱하여 획득).

## 핵심 제약사항

- 기술 스택: Android Native Java, MVVM + Repository + Retrofit2 (기존 아키텍처 유지)
- 인증: SessionManager + AuthInterceptor (Phase 1 완료) — userId 획득 경로 변경 없음
- 백엔드 API: `GET /jelly/user/{userId}` — 이미 JellyService/JellyRepository에 구현됨
- 상세 API: `GET /jelly/{jellyId}` — JellyResDTO 반환, 이미 JellyRepository에 구현됨

## 현재 코드의 문제점

1. **레이아웃 오류**: `LinearLayoutManager` 사용 → 3열 그리드가 아닌 1열 리스트로 표시됨
2. **아이템 클릭 미구현**: 탭 시 상세 페이지 이동 없음
3. **불필요한 복잡도**: `startAging` 콜백, 숙성 상태 배지 로직이 이 단계에서 섞여있음
4. **serverUrl 중복 선언**: JellyDrawerAdapter에 `"http://10.0.2.2:8080"` 하드코딩 (RetrofitClient.BASE_URL 미참조)

## 기능 요구사항

### 젤리서랍 목록 화면 (JellyDrawerActivity)
- 로그인된 userId로 `GET /jelly/user/{userId}` 호출
- 결과를 **3열 그리드** (GridLayoutManager(3))로 표시
- 세로 스크롤
- 각 아이템: 감정 아이콘 2개 (emo1Icon, emo2Icon) + 날짜 (createDate)
- 데이터 없을 때 빈 상태 메시지 표시
- userId가 없으면 에러 메시지 후 LoginActivity로 이동

### 젤리 상세 화면 (신규: JellyDetailActivity)
- 목록 아이템 탭 시 jellyId를 Intent Extra로 전달하여 진입
- `GET /jelly/{jellyId}` 호출하여 상세 정보 표시
- 표시 항목: jellyName, content, createDate
