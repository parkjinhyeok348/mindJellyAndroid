## Spacing Scale

All values are dp. Must be multiples of 4dp. No exceptions beyond the two noted below.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4dp | FrameLayout border padding in item_basic_emo (existing: 4dp — retain) |
| sm | 8dp | Badge horizontal padding (paddingHorizontal); inline label gaps |
| md | 12dp | item_jelly_drawer.xml root padding (existing: 12dp — retain) |
| lg | 16dp | Horizontal margins for drawer list (existing: 15dp left / 16dp right — standardise both to 16dp); EditText horizontal padding; button horizontal padding; emotion name margin-top |
| xl | 20dp | emoRecyclerView bottom margin (existing: 20dp — retain) |
| 2xl | 32dp | combinedJellyImageView bottom margin from grid (existing: 30dp — adjust to 32dp) |
| 3xl | 48dp | Save button bottom margin from screen edge |

Exceptions:
- 273dp top margin on rvJellyList is a background-image positional value, not a spacing token. Retain exactly as-is; it aligns with the `basic_background.png` canvas design.
- 53dp bottom margin on rvJellyList is a background-image positional value. Retain exactly as-is.
- Touch target minimum: 48dp height for all tappable Views (Material3 minimum).

## Status Badge Contract (DRAW-02)

The `tvAgingStatus` TextView uses a programmatic color swap in the Adapter's `onBindViewHolder` — not separate drawables. Three distinct states, driven by the `status` String field in `JellyDrawerResDTO`.

| Status value (backend) | Badge label (Korean) | Background color | Text color |
|------------------------|----------------------|------------------|------------|
| `"WAITING"` | 대기중 | `@color/badge_waiting_bg` | `@color/badge_waiting_text` |
| `"AGING"` | 숙성중 | `@color/badge_aging_bg` | `@color/badge_aging_text` |
| `"MATURED"` | 숙성완료 | `@color/badge_complete_bg` | `@color/badge_complete_text` |
| unknown / null | 대기중 (fallback) | `@color/badge_waiting_bg` | `@color/badge_waiting_text` |

Badge styling in XML (static, colors overridden programmatically):
- `android:paddingHorizontal="8dp"` (sm token)
- `android:paddingVertical="4dp"` (xs token)
- `android:textSize="12sp"` (Label role)
- `android:background` set to a rounded rectangle drawable `bg_status_badge.xml` (new drawable, 4dp corners, color set programmatically via `backgroundTintList`)

New drawable `res/drawable/bg_status_badge.xml`:
```xml
<?xml version="1.0" encoding="utf-8"?>
<shape xmlns:android="http://schemas.android.com/apk/res/android"
    android:shape="rectangle">
    <corners android:radius="4dp" />
    <solid android:color="#FFFFEEBA" />
</shape>
```
(The solid color is the default/waiting color; adapter overrides via `setBackgroundColor` or `DrawableCompat.setTint`.)

Implementation note: Use `holder.tvAgingStatus.setBackgroundColor(ContextCompat.getColor(context, badgeBg))` and `holder.tvAgingStatus.setTextColor(ContextCompat.getColor(context, badgeText))`. Requires `android:background="@drawable/bg_status_badge.xml"` to maintain rounded corners; use `GradientDrawable` mutate+setColor pattern to avoid tinting shared drawable state.

## Component Contracts

### 1. item_basic_emo.xml (emotion grid item — JELLY-01)

**Current state:** FrameLayout with emoImageView (60dp, 10dp margin, sel_jelly_item background).

**Change:** Add `tvEmoName` TextView below the image within the FrameLayout, or convert root to vertical LinearLayout.

**Recommended structure:** Convert root from `FrameLayout` to vertical `LinearLayout` (wrap_content × wrap_content). FrameLayout does not support vertical stacking cleanly.

```
LinearLayout (vertical, wrap_content × wrap_content)
  └── FrameLayout (wrap_content × wrap_content, background=@drawable/sel_jelly_item, padding=4dp)
        └── ImageView id=emoImageView (60dp × 60dp, layout_margin=10dp, layout_gravity=center)
  └── TextView id=tvEmoName
        width=match_parent (or wrap_content + centered)
        height=wrap_content
        textSize=12sp
        textColor=@color/text_secondary
        gravity=center
        layout_marginTop=4dp  [xs token]
        maxLines=1
        ellipsize=end
```

Sizing summary:
- Icon area: 60dp + 10dp margin each side = 80dp total FrameLayout inner size
- Name label: 12sp, centered below, 4dp top margin
- Total item height: ~100dp (80dp icon frame + 4dp gap + ~16dp label)
- Grid remains 3 columns — RecyclerView spanCount=3 unchanged

Selected state: existing `sel_jelly_item.xml` selector (bg_jelly_selected when selected, transparent otherwise) applies to the FrameLayout background. No change needed.

| State | rvJellyList | tvEmptyState + tvEmptyStateBody | pbLoading |

| Error | GONE | GONE | GONE (show Toast) |

## Implementation Notes for Executor

1. **colors.xml:** Add all 12 new color entries listed in the Color section before touching any layout XML.
2. **strings.xml:** Add all 21 string entries listed in the Copywriting Contract before touching any Java file.
3. **New drawables needed:** `bg_edit_text.xml` (EditText background), `bg_status_badge.xml` (badge rounded rect). Both defined in this spec.
4. **item_basic_emo.xml restructure:** Converting FrameLayout root to LinearLayout requires updating `BasicEmoAdapter.ViewHolder` field references — no functional change, just structural.
5. **ConstraintLayout height_percent:** Reduce `emoRecyclerView` constraint_height_percent from 0.7 to 0.5 to accommodate diary EditText + save button in the lower half of the screen.
6. **activity_jelly_drawer.xml root:** Converting from LinearLayout to FrameLayout enables overlay of list / empty state / progress without nested visibility hacks.
7. **Status badge color application:** Do NOT use `setBackgroundTintList` on a plain color background — it requires a drawable. Use `bg_status_badge.xml` as background, then call `mutate()` + `setColor()` on the `GradientDrawable` in `onBindViewHolder`.
8. **tvJellyId removal:** The ViewHolder field `tvJellyId` is removed. Confirm no other Activity/Fragment references this ID before deletion.
